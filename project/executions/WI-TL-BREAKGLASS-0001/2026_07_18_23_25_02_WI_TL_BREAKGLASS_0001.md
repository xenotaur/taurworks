# Summary

Implement WI-TL-BREAKGLASS-0001: simplify `tl`'s calling convention to
`tl NAME`, reframe it permanently as a break-glass fallback, unify the
`tl`/`tw`/`taurworks` README explanation, document the `project init`
`--local`/`--path` discoverability gap, and fix the legacy-admin activation
guidance that falsely claimed "was not sourced."

# Result

All five Required Changes implemented:

1. `sourceme/aliases.source`: `tl NAME` replaces `tl activate NAME` (usage
   message and header comment updated to permanent break-glass framing;
   lookup order and sourcing mechanics unchanged).
2. `README.md`: the old "Interim `tl` helper (temporary, feature-frozen)"
   section is replaced with one unified `taurworks`/`tw`/`tl` explanation,
   including the corrected regenerate-then-resource stale-shell-helper
   mitigation and a forward pointer to the in-design `tw install`/`refresh`.
3. `src/taurworks/cli.py`: `project init`'s `description=` now states that
   `--local`/`--path` are `create`-only.
4. `src/taurworks/project_resolution.py`: the legacy-admin guidance
   (previously an unconditional "was not sourced" claim) is now a 3-way
   message keyed on `legacy_sourcing_enabled`/`legacy_trusted` (both already
   computed by `_apply_legacy_trust_diagnostics` before this branch runs):
   Tier 1 disabled -> "was not sourced" (deterministically true regardless
   of shell-only state); enabled + trusted -> "will be sourced
   automatically" (also deterministic -- the trusted branch in
   `_tw_activate` never depends on `--legacy`/TTY/interactive choice);
   enabled + untrusted -> neutral wording, since that outcome genuinely
   depends on shell-only state this diagnostics call cannot see.
5. Tests: new `tests/aliases_source_test.py` (zero prior coverage) with six
   tests covering `tl NAME` sourcing from both the `Admin/` and
   `.taurworks/` script locations, the missing-script and no-argument error
   paths, and explicit rejection of the old two-argument
   `tl activate NAME` syntax. Added two regression tests to
   `tests/project_resolution_test.py` for the new 3-way guidance
   conditional (trusted+enabled; enabled+untrusted).

Nothing was skipped or deferred beyond what the WI's Non-Goals already
excluded (no `tl`-to-`tw` delegation, no `tw install`/`refresh`
implementation, no new `project init` flag).
