# Summary

Implement slices 4-5 of `WI-ACTIVATION-CONFIG-0001`: `taurworks legacy
inspect PROJECT` (conservative, read-only extraction of common patterns
from `Admin/project-setup.source`) and `taurworks legacy migrate PROJECT
[--apply]` (opt-in write of unambiguous detected patterns into
`.taurworks/config.toml`). Slice 6 (trusted hooks) is explicitly out of
scope per the work item's `## Out of Scope` section.

# Result

- New `src/taurworks/legacy.py`: `parse_legacy_setup_script` conservatively
  classifies each non-comment line of a legacy setup script into
  `conda_activate` / `export` / `cd` / `message` / `unsupported`. Literal-value
  detection rejects any value containing shell expansion/substitution markers
  (`$`, backticks, pipes, `&&`, `;`, etc.) as unsupported, matching the
  "conservative" mandate from the design doc. `gather_legacy_inspect_diagnostics`
  reads (never executes/sources) the script and reports counts plus per-line
  detections; `format_legacy_inspect_output` always redacts export values,
  reusing the existing `activation.exports` redaction convention from
  `project activate --print` rather than inventing a new heuristic (resolves
  the WI's Open Question on this point).
- `gather_legacy_migrate_diagnostics` computes an unambiguous config patch:
  a field is only applied when there is exactly one non-conflicting detection
  for it and the field is not already configured in `.taurworks/config.toml`;
  conflicts (duplicate exports, multiple `conda activate`/message/`cd` lines),
  already-configured fields, and unresolvable `cd` targets are recorded as
  `manual_review`/`skipped` rather than applied. Without `--apply`, nothing is
  written (dry run). With `--apply`, the patch is merged into the existing
  config via `project_internals.ensure_minimal_project_config` /
  `write_project_config`, creating `.taurworks/` if needed.
- `cd` lines are mapped to `paths.working_dir` only when the path is a
  literal, existing relative path under the project root (validated via
  `project_internals.relative_working_dir_metadata`); otherwise flagged for
  manual review.
- `src/taurworks/cli.py`: added a `legacy` subparser namespace (`legacy
  inspect PROJECT`, `legacy migrate PROJECT [--apply]`) and a
  `_handle_legacy_command` dispatcher, following the existing
  `gather_X_diagnostics`/`format_X_output` convention used throughout
  `project_resolution.py`.
- `tests/legacy_test.py` (18 tests): pattern detection for all four kinds,
  redaction of export values in both the raw match data's intent
  (`value_redacted` flag) and the rendered output, unsupported/dynamic value
  rejection, invalid export name handling, read-only guarantee for `inspect`
  (no filesystem mutation), dry-run vs. `--apply` behavior, ambiguous
  duplicate-export handling, out-of-project-root `cd` handling, and
  preservation of pre-existing config values on `--apply`.
