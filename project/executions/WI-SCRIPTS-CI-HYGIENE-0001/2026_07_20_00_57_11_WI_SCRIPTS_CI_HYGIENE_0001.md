# Summary

Implement `WI-SCRIPTS-CI-HYGIENE-0001`: close the scripts/ and CI hygiene
gaps found by the LRH-benchmarked audit — standalone tool-version
reporting, workflow YAML validation wired into CI, and `scripts/build`/
`scripts/clean` dev hygiene.

# Result

Added four new scripts, all executable, matching the bash conventions of
the existing `scripts/*` entries:

- `scripts/version` — prints `python`, `pip`, `black`, `ruff` versions
  standalone (no longer only a side effect of `scripts/develop`).
- `scripts/check-workflows` — validates `.github/workflows/*.yml` syntax
  via `yaml.safe_load`, exits non-zero on invalid YAML.
- `scripts/build` — `python setup.py sdist bdist_wheel` into `dist/`, no
  new build-tool dependency (`setup.py`/setuptools/wheel already present).
- `scripts/clean` — removes `dist/`, `build/`, `src/taurworks.egg-info`,
  `__pycache__`, `.ruff_cache`.

Edited `.github/workflows/python-ci.yml`: added "Check workflows" and
"Report tool versions" steps ahead of Lint. `scripts/develop` unchanged,
remains setup/bootstrap-only.

**Discovered mid-implementation, decided with the user (AskUserQuestion,
confirmed):** genuine YAML syntax validation requires a YAML parser, and
PyYAML was not available anywhere in taurworks's own toolchain — not in
the Taurworks conda env, not in `constraints-dev.txt`, not vendored by
`pip`/`setuptools`, and CI's `actions/setup-python` would not have it
either (only base anaconda has it, as an unrelated transitive dependency
not available to a clean checkout or CI). The WI's Required Change #2 said
"do not add a new dependency," but a stdlib-only heuristic checker would
not be real YAML validation and would also need to live outside `scripts/`
per `STYLE.md` Rule 8 (thin scripts, no core logic). User chose: add
`pyyaml==6.0.3` as a pinned dev dependency (`constraints-dev.txt`,
`setup.py` `dev` extras) over the heuristic-checker and
drop-the-script alternatives.

Verified `scripts/publish`'s "No dist directory found, please run build
first" message: after `scripts/build`, `scripts/publish` reaches the real
twine-upload echo branch instead.
