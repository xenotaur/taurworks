# Summary

Address four open review comments on PR #69 (WI-ACTIVATION-PRODUCERS-0001
implementation): two Codex P2 comments and two Copilot comments, all

# Result

All four comments verified valid by code inspection before fixing; all
fixed (commit 2055dd0):

1. **Codex P2 + Copilot (same root cause) — `env set` couldn't repair
   malformed activation metadata.** `set_activation_environment` called
   `activation_environment_from_config(config)` on the *existing* value
   before writing the new one, so a broken existing
   `[activation.environment]` (bad type, missing name) raised
   `ProjectConfigError` and blocked the explicit setter from ever fixing
   it. Now wrapped in try/except; malformed existing data is treated as
   "no previous environment" and overwritten, matching the setter's
   purpose.
2. **Codex P2 — `env set --project NAME` guidance fails for registered/
   workspace projects.** `gather_project_env_set/show_diagnostics` used
   the narrow `project_internals.resolve_project_target` (cwd/path only),
   while the guidance text this PR added
   (`taurworks project env set ENV_NAME --project NAME`) is generated by
   `gather_project_activate_print_diagnostics`, which resolves via the
   full registry/workspace-aware `resolve_global_activation_project`. For
   a registered project outside the cwd hierarchy, the suggested command
   failed with "No Taurworks project metadata found." Both `env set` and
   `env show` now use `resolve_global_activation_project`, matching
   `activate`'s resolution exactly.
3. **Copilot — `_write_initial_project_config` could crash
   refresh/create.** It called `read_project_config` unguarded; invalid
   TOML or an unreadable file raised uncaught, crashing
   `manager.refresh_project`/`create_project` — a new crash surface this
   PR introduced (these functions never parsed config.toml before), and a
   direct contradiction of the function's own docstring promise to leave
   malformed config untouched. Now wrapped in try/except, returning no
   changes on failure.

Nothing was skipped.
